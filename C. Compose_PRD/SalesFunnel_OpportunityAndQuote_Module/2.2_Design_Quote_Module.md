# Design Spec — Quotation Module UI
**Product:** UT Service Console  
**Module:** Quote Detail Page  
**Version:** 1.0  
**Last Updated:** 2026-06-11  
**Design System Reference:** `Design_System.md` v1.0

---

## 1. Design Principles for this Module

- **Simpler than Opportunity** — the Quote page is a focused transactional form, not a dashboard. It has fewer panels, less chrome.
- **Relation is always visible** — the link back to the parent Opportunity is persistent and prominent in the header area.
- **Tab structure separates metadata from line items** — Quote details live in one tab; product/part entry lives in another. This keeps the form approachable and prevents scroll fatigue.
- **State drives layout** — what the user can do changes based on Quote status. Controls appear, hide, or go read-only based on `Status`.

## 2. Page Layout

### 2.1 Overall Structure

```
┌─────────────────────────────────────────────────────────────────────┐
│  [Breadcrumb: Opportunities > Opp Name > Quote Q-2025-001 v2]       │
│  [Quote Header: Number · Version badge · Status badge · Actions]    │
├─────────────┬───────────────────────────────────────────────────────┤
│             │                                                         │
│  Left Panel │  [Tab Bar: Details | Quote Items | Version History]    │
│  (Related   │  ─────────────────────────────────────────────────    │
│   Links)    │  [Active Tab Content]                                  │
│             │                                                         │
└─────────────┴───────────────────────────────────────────────────────┘
```

- **Left panel** — fixed width `w-64` (256px); contains Related Links only (see §4)
- **Right content area** — flex-1; contains tab bar + tab content
- **No nested sidebars** — this page intentionally avoids the deep three-column layouts used in Work Order detail

## 6. Tab 1 — Details

The Details tab is a two-column form grid within a `.card`.

### 6.1 Field Groups

**Group A — Quote Information**

| Field | Input Type | Edit State | Notes |
|-------|-----------|-----------|-------|
| Quote Name | Text input | Draft only | Auto-populated; editable |
| Status | Read-only badge | Never editable | Driven by workflow |
| Version | Read-only text | Never editable | e.g., `Version 2` |
| Expiration Date | Date picker | Draft only | Required |
| Currency | Read-only | Never editable | Inherited from Opportunity |

**Group B — Linked Records** *(read-only in all states)*

| Field | Display | Notes |
|-------|---------|-------|
| Opportunity | Link + name | `var(--color-info)` |
| Account | Link + name | `var(--color-info)` |
| Contact | Link + name | `var(--color-info)` |
| Pricebook | Text | Read-only |

**Group C — Pricing Summary** *(read-only, auto-calculated)*

| Field | Display | Notes |
|-------|---------|-------|
| Subtotal | Currency | From line items |
| Discount | Percentage | Weighted average |
| Tax | Currency input | Editable in Draft |
| Shipping & Handling | Currency input | Editable in Draft |
| Grand Total | Currency, `font-bold text-lg` | Calculated |

**Group D — Approval Information** *(visible only after Submitted)*

| Field | Display | Visible When |
|-------|---------|-------------|
| Submitted By | User name + avatar | Status ≥ Submitted |
| Submitted Date | Formatted datetime | Status ≥ Submitted |
| Reviewed By | SDH name | Status = Approved or Rejected |
| Review Date | Formatted datetime | Status = Approved or Rejected |
| Approval Comments | Textarea (read-only) | Status = Approved |
| Rejection Reason | Textarea (read-only), `var(--color-danger)` left border | Status = Rejected |

**Group E — Notes / Description**

| Field | Input Type | Edit State |
|-------|-----------|-----------|
| Internal Notes | Textarea | Draft only |

### 6.2 Form Field Layout

Two columns using CSS grid:

```jsx
className="grid grid-cols-2 gap-4"
```

Full-width fields (Notes, Rejection Reason, Approval Comments) use:

```jsx
className="col-span-2"
```

### 6.3 Section Dividers

Use a horizontal rule between field groups:

```jsx
<div className="col-span-2 border-t my-2" style={{ borderColor: 'var(--border)' }} />
```

Section label above each group:

```jsx
className="col-span-2 text-[11px] font-bold uppercase tracking-wider"
style={{ color: 'var(--text-muted)' }}
```

## 7. Tab 2 — Quote Items

This is the primary data entry tab. It is a full-width table with an add/edit interaction.

### 7.1 Header Row (above table)

```
[Quote Items]                              [+ Add Item]  (Draft only)
Subtotal: $12,400.00   Discount: 5%   Tax: $620.00   Grand Total: $13,020.00
```

- Totals row uses `text-sm font-medium` `var(--text-secondary)` with pipe separators
- Grand Total is `font-bold` `var(--text-main)`

### 7.2 Line Items Table

```
┌──────┬──────────────────┬────────┬──────────┬────────────┬──────────┬──────────┬───┐
│  #   │  Product / Part  │  SKU   │   Qty    │ Unit Price │ Disc (%) │  Total   │   │
├──────┼──────────────────┼────────┼──────────┼────────────┼──────────┼──────────┼───┤
│  1   │ HVAC Filter Set  │ HV-001 │    4     │  $250.00   │   10%    │  $900.00 │ ✎ │
│  2   │ Labour - 4hr     │ LB-004 │    1     │ $1,200.00  │    0%    │$1,200.00 │ ✎ │
└──────┴──────────────────┴────────┴──────────┴────────────┴──────────┴──────────┴───┘
```

**Column Widths:**

| Column | Width | Alignment |
|--------|-------|-----------|
| # | `w-10` | Center |
| Product / Part | `flex-1` min `w-48` | Left |
| SKU | `w-32` | Left |
| Qty | `w-20` | Right |
| Unit Price | `w-32` | Right |
| Disc (%) | `w-24` | Right |
| Total | `w-32` | Right, `font-medium` |
| Actions | `w-16` | Center |

### 7.3 Table Styles

```jsx
// Table header
className="text-[11px] font-bold uppercase tracking-wider px-3 py-2 text-left"
style={{ color: 'var(--text-muted)', backgroundColor: 'var(--bg-light)', borderBottom: '1px solid var(--border)' }}

// Table row
className="border-b transition-colors"
style={{ borderColor: 'var(--border)' }}
// Row hover:
style={{ backgroundColor: 'var(--bg-light)' }}

// Table cell
className="px-3 py-3 text-sm"
style={{ color: 'var(--text-secondary)' }}
```

### 7.4 Inline Row Editing (Draft State Only)

Clicking the edit icon (✎) on a row switches that row to **inline edit mode**:

- Product/Part → `<input>` with product search/autocomplete (`.input-field`)
- SKU → `<input>` text (`.input-field`)
- Qty → `<input type="number">` (`.input-field` `w-16 text-right`)
- Unit Price → `<input type="number">` currency (`.input-field` `w-28 text-right`)
- Disc (%) → `<input type="number">` 0–100 (`.input-field` `w-16 text-right`)
- Total → Calculated, still read-only, updates live as Qty / Price / Disc change

Row actions in edit mode:

```jsx
// Save row
className="p-1.5 rounded text-xs font-medium"
style={{ backgroundColor: 'var(--accent)', color: '#1a1a1a' }}

// Cancel row edit
className="p-1.5 rounded text-xs font-medium"
style={{ backgroundColor: 'var(--bg-light)', color: 'var(--text-secondary)', border: '1px solid var(--border)' }}

// Delete row (only in edit mode)
className="p-1.5 rounded"
style={{ color: 'var(--color-danger)' }}
```

## 9. Rejection Modal (SDH Action)

### 9.1 Modal Structure

```
┌──────────────────────────────────────────────┐
│  Reject Quote                           [✕]  │
├──────────────────────────────────────────────┤
│  Quote: Q-2025-001 v2                        │
│  Opportunity: Acme Corp — HVAC Package       │
│  Grand Total: $13,020.00                     │
│                                              │
│  Rejection Reason *                          │
│  ┌────────────────────────────────────────┐  │
│  │  Describe why this quote is being      │  │
│  │  rejected so the rep can revise it.    │  │
│  └────────────────────────────────────────┘  │
│  * Required — minimum 10 characters          │
│                                              │
│  [Cancel]              [Confirm Rejection]   │
└──────────────────────────────────────────────┘
```

- Modal max-width: `max-w-lg`
- Textarea: `.input-field` rows=4; shows red border + error message if submitted empty
- "Confirm Rejection" button: `.btn-primary` with `var(--color-danger)` background (override accent for this destructive action only)
- "Cancel" button: `.btn-secondary`

## 11. Responsive Behaviour

| Breakpoint | Layout Change |
|-----------|--------------|
| `xl` (1280px+) | Full two-column layout (left panel + content) |
| `lg` (1024px–1279px) | Left panel collapses to an icon-only strip |
| `md` (768px–1023px) | Left panel hidden; accessible via "Related" tab added to tab bar |
| `< md` | Single column; all tabs stacked |
