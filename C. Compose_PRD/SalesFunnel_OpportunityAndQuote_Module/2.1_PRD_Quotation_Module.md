# PRD — Quotation Module
**Product:** UT Service Console  
**Module:** Quotation (Quote)  
**Version:** 1.0  
**Last Updated:** 2026-06-11  
**Status:** Draft  
**Owner:** Product Team

---

## 1. Overview

The Quotation Module enables field service and sales operations teams to generate, manage, negotiate, and approve price quotes that are directly linked to Opportunities in the UT Service Console. A Quote captures proposed service products and parts at negotiated prices, is reviewed by the SDH (Service Delivery Head) for approval, and — once approved — automatically updates the corresponding Opportunity's line item prices.

This PRD covers the **full lifecycle** of a Quote: creation from an Opportunity, line item entry, SDH-level approval, rejection and re-submission flow (Quote v2+), and final price write-back to the Opportunity.

---

## 2. Problem Statement

Currently, price negotiation during the Opportunity lifecycle is handled informally — over email or verbal discussion — with no structured record in the system. This creates:

- No audit trail for negotiated pricing decisions
- No approval gate before prices are committed to an Opportunity
- No version history when quotes are revised after rejection
- Disconnected data between proposed prices and actual Opportunity values

The Quotation Module solves this by embedding a formal Quote workflow inside the Opportunity lifecycle, governed by an SDH approval step.

---

## 3. Goals

| # | Goal |
|---|------|
| G1 | Allow creation of a Quote from an Opportunity once the Opportunity reaches the **Negotiation** phase |
| G2 | Support entry of service products and parts as Quote Line Items with quantity, unit price, and discount |
| G3 | Route submitted Quotes to the SDH role for approval |
| G4 | Support Quote rejection and re-submission as a revised version (Quote v2, v3…) |
| G5 | On approval, write final negotiated prices back to the Opportunity's line items |
| G6 | Maintain a full version history of all Quote submissions per Opportunity |

---

## 4. Non-Goals

- This module does not replace a full CPQ (Configure-Price-Quote) tool; it is purpose-built for UT Service Console's field service context.
- This module does not handle customer-facing PDF generation in v1 (see `QuoteDocument` in Salesforce API; may be added in v2).
- This module does not manage purchase orders, invoicing, or payment collection.

---

## 5. User Roles & Permissions

| Role | Permissions |
|------|-------------|
| **Field Agent / Sales Rep** | Create Quote, add/edit Line Items, submit Quote for approval, submit revised Quote (v2+) |
| **SDH (Service Delivery Head)** | View all Quotes pending approval, approve or reject, add rejection comments |
| **Admin** | Full access; can override any Quote status |
| **Read-Only / Other Roles** | View Quote details and history only |

## 6. Opportunity Integration — Entry Point

### 6.1 Chevron Progress Bar Trigger

The Opportunity record uses a **chevron progress bar** to display lifecycle phases. The standard phases are:

```
[Prospect] → [Qualification] → [Proposal] → [Negotiation] → [Closed Won / Closed Lost]
```

A **"Submit Negotiation"** action becomes available when the Opportunity chevron is on the **Negotiation** phase. Clicking this action does not submit the Opportunity itself — it creates (or opens) the linked Quote for that Opportunity.

### 6.2 Opportunity ↔ Quote Relationship

- One Opportunity may have **many Quotes** (versions) over time, but only **one active/pending Quote** at a time.
- Quote stores `OpportunityId` (FK to Opportunity).
- The `IsSyncing` flag on the Salesforce Quote object tracks whether the active approved Quote is synced back to the Opportunity.

### 6.3 Opportunity Quote Status Indicator

On the Opportunity record, a read-only field shows the current Quote state:

| Quote State | Indicator Label |
|-------------|----------------|
| No Quote yet | — |
| Draft | `Quote Draft` |
| Submitted (Pending SDH) | `Awaiting Approval` |
| Rejected | `Revision Required` |
| Approved | `Quote Approved` |

## 7. Quote Lifecycle

```
                    ┌──────────────────┐
                    │   Opportunity    │
                    │  (Negotiation    │
                    │    Phase)        │
                    └────────┬─────────┘
                             │  "Submit Negotiation" action
                             ▼
                    ┌──────────────────┐
                    │  Create Quote    │◄──────────────┐
                    │    (Draft)       │               │
                    └────────┬─────────┘               │
                             │  Add Line Items          │
                             │  Fill Quote Details      │
                             ▼                         │
                    ┌──────────────────┐               │
                    │  Submit Quote    │               │  Rejection → v2
                    │  (Pending SDH    │               │
                    │   Approval)      │               │
                    └────────┬─────────┘               │
                             │                         │
               ┌─────────────┴─────────────┐          │
               ▼                           ▼          │
      ┌─────────────────┐       ┌──────────────────┐  │
      │   SDH Approves  │       │  SDH Rejects     │──┘
      └────────┬────────┘       │  (with comment)  │
               │                └──────────────────┘
               ▼
      ┌─────────────────┐
      │  Write prices   │
      │  back to Opp.   │
      │  Line Items     │
      └─────────────────┘
```

### 7.1 Phase Definitions

| Phase | Description |
|-------|-------------|
| **Draft** | Quote is created; line items being entered; not yet submitted |
| **Submitted** | Quote has been submitted by the rep; awaiting SDH action |
| **Pending Approval** | System routing state while SDH has not yet acted |
| **Approved** | SDH has approved; prices are being written back to the Opportunity |
| **Rejected** | SDH has rejected; rep must create a revised Quote (v2) |
| **Superseded** | A newer version of this Quote has been submitted (v2 replaces v1) |

## 8. Functional Requirements

### 8.1 Quote Creation

| ID | Requirement |
|----|-------------|
| F-01 | The system SHALL allow creation of a Quote from the Opportunity detail page when the Opportunity is in the **Negotiation** phase |
| F-02 | Quote creation SHALL auto-populate: Account, Contact, Opportunity reference, and Currency from the parent Opportunity |
| F-03 | Only one non-superseded Quote SHALL be active per Opportunity at a time |
| F-04 | Draft Quotes SHALL be editable by the creating rep until submitted |

### 8.2 Quote Line Items

| ID | Requirement |
|----|-------------|
| F-05 | Reps SHALL be able to add Line Items representing service products or parts |
| F-06 | Each Line Item SHALL capture: Product/Part Name, SKU/Code, Quantity, Unit Price, Discount (%), Line Total |
| F-07 | The system SHALL auto-calculate Line Total = (Quantity × Unit Price) × (1 − Discount) |
| F-08 | The system SHALL display Quote Subtotal, Tax (editable), Shipping & Handling, and Grand Total |
| F-09 | Line Items SHALL be addable, editable, and deletable while the Quote is in Draft state |
| F-10 | Once submitted, Line Items SHALL be read-only until a rejection prompts a new version |

### 8.3 Quote Submission

| ID | Requirement |
|----|-------------|
| F-11 | Rep SHALL be able to submit a Draft Quote via a **"Submit for Approval"** action |
| F-12 | Submission SHALL validate that at least one Line Item exists before proceeding |
| F-13 | On submission, Quote status changes to **Submitted / Pending Approval** |
| F-14 | The SDH role SHALL receive an in-app notification upon Quote submission |
| F-15 | A submission timestamp and submitter name SHALL be recorded |

### 8.4 SDH Approval

| ID | Requirement |
|----|-------------|
| F-16 | Users with the SDH role SHALL see a dedicated **Pending Approvals** queue |
| F-17 | SDH SHALL be able to view the full Quote detail including all Line Items before deciding |
| F-18 | SDH SHALL be able to **Approve** a Quote, with optional comments |
| F-19 | SDH SHALL be able to **Reject** a Quote and MUST provide a rejection reason (required field) |
| F-20 | Approval/rejection action SHALL be timestamped and attributed to the approver |

### 8.5 Rejection & Re-submission (Quote v2)

| ID | Requirement |
|----|-------------|
| F-21 | On rejection, the rep SHALL receive an in-app notification with the SDH's rejection reason |
| F-22 | The rep SHALL be able to create a **Revised Quote** (new version number) pre-populated with the previous Quote's line items |
| F-23 | The previous rejected version SHALL be retained in the version history as **Rejected** |
| F-24 | The revised Quote SHALL follow the same Draft → Submitted → Approved/Rejected lifecycle |
| F-25 | Version numbering SHALL be system-managed and displayed (e.g., "Quote #Q-2025-001 v2") |

### 8.6 Approval → Opportunity Price Write-back

| ID | Requirement |
|----|-------------|
| F-26 | On SDH approval, the system SHALL automatically update the parent Opportunity's line item prices with the approved Quote's line item prices |
| F-27 | The Opportunity's `Amount` / price fields SHALL reflect the approved Grand Total |
| F-28 | `IsSyncing` flag on the Quote SHALL be set to `true` upon approval |
| F-29 | The Opportunity SHALL display a read-only "Price Updated from Quote [ID]" notice |
| F-30 | Price write-back SHALL be logged in the Opportunity's activity history |

## 11. Business Rules

1. A Quote can only be created from an Opportunity in the **Negotiation** phase or later.
2. A Quote cannot be submitted with zero line items.
3. Only one Quote can be in **Submitted / Pending Approval** state per Opportunity at any time.
4. Creating a v2 Quote automatically marks the previous version as **Superseded** (if it was rejected).
5. SDH cannot approve or reject their own submitted quotes (conflict of interest guard).
6. Approved Quote prices override existing Opportunity line item prices — this action is irreversible through the UI (admin override only).
7. An Opportunity can be moved out of Negotiation without a Quote if the deal is closed-lost.

## 12. Out of Scope for v1

- PDF generation / emailing Quote to customer
- Multi-currency Quote line item mixing
- Customer-facing Quote portal / e-signature
- Automated tax calculation via third-party provider
- Bulk approval queue for SDH
- Scheduled Quote expiry notifications
